# @versemate/visuals

Versemate per-book visual aids — BibleProject Read Scripture posters, BibleProject animated overview videos, Precept Austin chapter charts, Insight for Living structural charts, and Versemate originals.

Shared by **verse-mate-web** (Vite SPA) and **verse-mate-mobile** (Expo React Native).

## Usage

```ts
import {
  getVisualsForBook,
  getCardsForChapter,
  getVideoForChapter,
  absolutizeVisualUrl,
  BOOKS_WITH_VISUALS,
} from '@versemate/visuals';

// Tab-visibility gate — keeps the registry off the critical render path.
if (BOOKS_WITH_VISUALS.has(slug)) {
  // show Visuals tab
}

const manifest = getVisualsForBook('genesis');
const cards = getCardsForChapter(manifest, 1);
const video = getVideoForChapter(manifest, 1);

// React Native: absolutize before passing to <Image source={{ uri }} />
const uri = absolutizeVisualUrl(cards[0].thumb);
// → https://app.versemate.org/visuals/genesis/precept_chapter_1.png
```

## Path convention

Registry image paths are **root-relative** (`/visuals/<slug>/<file>`). The web app serves them directly from its `public/` directory. Mobile prefixes them with `VISUALS_ORIGIN` (`https://app.versemate.org`) via `absolutizeVisualUrl()`.

This keeps a single source of truth and lets the web app's CDN remain canonical for both clients.

## Coverage

- 66 books (every canonical book has at least the BibleProject poster + overview video where verified)
- ~2,000 curated images sourced from Precept Austin + BibleProject + Insight for Living
- All Precept Austin charts attributed CC BY-SA 4.0 (BibleProject) / "Insight for Living Ministries" (Swindoll)

## Authoring pipeline

Content is generated by `verse-mate-web/scripts/visuals-ingest/build_manifests.py`, which:

1. Walks `verse-mate-web/public/visuals/<slug>/` to find ingested assets.
2. Writes `verse-mate-web/src/data/visuals/registry.ts` (web bundle).
3. Writes `verse-mate-web/src/data/visuals/booksWithVisuals.ts` (light slug-set for tab gating).
4. Writes `verse-mate-visuals/src/registry.ts` (this package) with identical content.

Run after adding new assets:

```sh
cd verse-mate-web
python3 scripts/visuals-ingest/build_manifests.py
```

## Why a separate package?

Mirrors `@versemate/studies`: shared data, two clients, zero duplication. Web's Vite + mobile's Metro both treeshake from the same TS module. ~500KB of registry data — fine as a single file without per-chapter dynamic splitting (unlike studies, where each chapter is its own chunk to stay under Cloudflare Workers' 25 MiB asset limit).

If the registry grows past a few MB, we'll migrate it to JSON + API-served fetches (the package's shape stays canonical regardless of transport).
