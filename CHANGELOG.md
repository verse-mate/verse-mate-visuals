# Changelog

All notable changes to `@versemate/visuals` are documented here. Format adapted to a generated-data package: structural changes (shape, helpers, public API) get full entries; routine registry regenerations get summary lines.

## [0.1.0] — 2026-05-19

### Features

- Initial public surface: `VISUALS_REGISTRY`, `BOOKS_WITH_VISUALS`, `getVisualsForBook`, `getCardsForChapter`, `getVideoForChapter`, `absolutizeVisualUrl`, `VISUALS_ORIGIN`
- Types: `VisualCard`, `VideoEntry`, `VisualsManifest` (also exported from `./types`)
- Path convention: registry stores root-relative paths; `absolutizeVisualUrl()` prefixes `https://app.versemate.org` for React Native consumers
- Chapter-aware filtering: `getCardsForChapter` honors per-card `chapters?: number[]` for Precept Austin per-chapter charts; `getVideoForChapter` honors `chapterStart`/`chapterEnd` for multi-part BibleProject overviews (Genesis 1–11, Genesis 12–50, …)

### Content

- 66 books (all canonical OT + NT)
- ~2,000 curated images after audit + cross-book MD5 dedup
- Sources: BibleProject Read Scripture posters (CC BY-SA 4.0), BibleProject overview videos, Precept Austin commentary charts (Bruce Hurt), Insight for Living Swindoll structural charts, VerseMate originals (James)

### Tooling

- Generated by [`scripts/visuals-ingest/build_manifests.py`](https://github.com/verse-mate/verse-mate-web/blob/main/scripts/visuals-ingest/build_manifests.py) in verse-mate-web
- Web app retains its own local copy at `src/data/visuals/registry.ts`; this package is the source-of-truth for verse-mate-mobile via Metro
